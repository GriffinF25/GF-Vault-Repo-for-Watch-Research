## How to Use This System

### Starting a Task

1. **Read AGENTS.md** — Understand your autonomy and boundaries
2. **Check INDEX.md** — Identify which domain owns this work
3. **Read domain CLAUDE.md** — Understand domain strategy
4. **Check domain `/memory/`** — See prior decisions
5. **Work in domain `/projects/`** — Execute the work
6. **Update domain `/memory/`** — Record learnings before finishing

### Finding Information

| Need | Where to Look |
|------|---------------|
| Domain strategy | `[domain]/CLAUDE.md` or `[domain]/memory/` |
| Workspace principles | `SYSTEM.md` or `AGENTS.md` |
| Domain registry | `INDEX.md` |
| Memory access | `MEMORY.md` for index; actual memory in `[domain]/memory/` |
| Reusable templates | `/_shared/templates/` |
| Frameworks | `/_shared/frameworks/` |

### Updating Memory

**When to save:**
- Strategic decisions made
- Significant learnings discovered
- New frameworks or patterns developed
- Cross-domain bridges documented

**How to save:**
1. Create file in `[domain]/memory/domain-topic.md`
2. Include frontmatter: name, description, type
3. Use `[[domain-topic]]` links for cross-domain references

### Cross-Domain Work

When work spans multiple domains:
1. Identify all affected domains
2. Review each domain's strategy in CLAUDE.md
3. Update memory in each affected domain
4. Use cross-domain links to connect decisions
