## Adding to Shared Resources

When to promote work to `/_shared/`:
1. Used by 2+ domains successfully
2. General enough for reuse
3. Stable and well-documented
4. Reduces duplication

**Process:**
1. Document the template/framework
2. Test in originating domain
3. Get feedback from other domains
4. Move to appropriate `/shared/` subfolder
5. Update this README

## Maintenance

### Monthly
- Review templates for continued relevance
- Update based on domain feedback
- Archive outdated materials

### Quarterly
- Consolidate similar templates
- Refactor frameworks based on usage
- Update style guides
